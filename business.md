
# BeautyTrack — CRM для косметологов с визуальным трекингом
## Техническое задание

---

## 1. Функциональные требования (FR)

### Модуль 1: Карточка клиента и визуальный трекинг (ядро)

| ID | Требование | Приоритет |
|----|------------|------------|
| FR-01 | Система должна позволять создавать профиль клиента с полями: ФИО, дата рождения, контакты (телефон, email), аллергии, противопоказания | Must |
| FR-02 | Система должна хранить **галерею фото «До/После»** с возможностью загрузки до 10 фото за визит | Must |
| FR-03 | Система должна автоматически **накладывать маску лица** (точки ориентиры: глаза, нос, губы) для точного сравнения снимков в разных ракурсах | Must |
| FR-04 | Система должна отображать **динамику изменений** в виде графика по параметрам: глубина морщин, пигментация, тон кожи, поры (на основе ИИ-анализа фото) | Should |
| FR-05 | Система должна позволять врачу **аннотировать фото** (рисовать стрелки, круги, добавлять текст: «зона коррекции») | Could |
| FR-06 | Система должна генерировать **PDF-отчет о прогрессе** за 3/6/12 месяцев для отправки клиенту | Should |

### Модуль 2: Онлайн-запись 24/7

| ID | Требование | Приоритет |
|----|------------|------------|
| FR-07 | Система должна предоставлять клиенту **публичную страницу записи** (виджет для вставки на сайт или отдельная ссылка) | Must |
| FR-08 | Клиент должен видеть **свободные слоты** в реальном времени с учетом: график врача, длительность процедуры, перерывы | Must |
| FR-09 | Система должна **блокировать конфликтующие записи** (одновременная запись двух клиентов на одного врача) | Must |
| FR-10 | Клиент должен иметь возможность **отменить/перенести запись** онлайн за N часов до визита (настраивается: 2/6/12/24ч) | Must |
| FR-11 | Система должна автоматически **отправлять ссылку на предзаполненную анкету** (аллергии, согласие на процедуры) после записи | Could |
| FR-12 | Система должна поддерживать **запись пакетом процедур** (например: 3 лазерные эпиляции с интервалом 4 недели) | Should |

### Модуль 3: Автоподбор уходовой косметики

| ID | Требование | Приоритет |
|----|------------|------------|
| FR-18 | Система должна содержать **базу косметических средств** (INCI-состав, тип кожи, проблемы решает, противопоказания) | Must |
| FR-19 | Система должна на основе **типа кожи** (сухая/жирная/комби/норм) и **проблем** (акне, пигментация, купероз) предлагать **топ-3 средства** из базы | Must |
| FR-20 | Врач должен иметь возможность **корректировать рекомендацию** вручную (добавлять/убирать продукты) | Must |
| FR-21 | Система должна генерировать **персональную PDF-схему ухода** (утро/вечер, порядок нанесения, частота) для отправки клиенту | Must |
| FR-22 | Система должна учитывать **аллергии и противопоказания** клиента при подборе (исключать продукты с компонентом X) | Should |
| FR-24 | Система должна позволять врачу **загружать свои продукты** (кастомная косметика клиники) с ручным заполнением свойств | Should |

### Модуль 5: Администрирование и отчеты

| ID | Требование | Приоритет |
|----|------------|------------|
| FR-25 | Система должна показывать дашборд с KPI: загрузка врачей, отмены, выручка, средний чек | Must |
| FR-26 | Система должна экспортировать отчеты (Excel/CSV) по записям, клиентам, финансам | Should |
| FR-27 | Система должна иметь ролевую модель: администратор (полный доступ), врач (свои клиенты), клиент (только себя) | Must |

---

## 3. MVP (Минимально жизнеспособный продукт)

**Release 1.0 (2-3 месяца):**
- ✅ Профиль клиента + галерея фото (только ручное сравнение)
- ✅ Онлайн-запись с календарем (без пакетов)
- ✅ Напоминания по SMS + Telegram
- ✅ База косметики из 100 готовых средств (ручной ввод врачом)
- ✅ PDF-схема ухода
