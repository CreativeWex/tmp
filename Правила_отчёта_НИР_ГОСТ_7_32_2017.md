# Правила составления отчёта НИР по ГОСТ 7.32-2017

> Документ составлен на основе анализа финального отчёта о НИР «Разработка и исследование
> программной системы интеллектуального скоринга стартапов с применением AI-агентов».
> Правила применимы к любому программному приложению без изменения структуры.

---

## 1. ОБЯЗАТЕЛЬНАЯ ПОСЛЕДОВАТЕЛЬНОСТЬ СТРУКТУРНЫХ ЭЛЕМЕНТОВ

Каждый элемент начинается с новой страницы. Порядок строго фиксирован:

| № | Элемент | Обязательность |
|---|---|---|
| 1 | Титульный лист | обязательно |
| 2 | Список исполнителей | при >1 исполнителе |
| 3 | Реферат | обязательно |
| 4 | Содержание | обязательно |
| 5 | Перечень обозначений и сокращений | при наличии |
| 6 | Введение | обязательно |
| 7 | Основная часть (нумерованные разделы) | обязательно |
| 8 | Заключение | обязательно |
| 9 | Список использованных источников | обязательно |
| 10 | Приложения | при наличии |

## 2. ТИТУЛЬНЫЙ ЛИСТ

Строки сверху вниз в строго фиксированном порядке:

```
УДК XXX.X:XXX.X
[наименование организации]
[гриф утверждения — правый верхний угол]

ОТЧЁТ
О НАУЧНО-ИССЛЕДОВАТЕЛЬСКОЙ РАБОТЕ
по теме:
«НАЗВАНИЕ РАБОТЫ ПРОПИСНЫМИ БУКВАМИ»

Город, Год
```

### Правила выбора УДК

Подбирается по тематике разработки. Наиболее частые коды:

| Тематика | Код УДК |
|---|---|
| Искусственный интеллект | 004.8 |
| Прикладные программные системы | 004.9 |
| Базы данных | 004.6 |
| Разработка ПО | 004.4 |
| Веб-приложения | 004.774 |
| Машинное обучение | 004.85 |

Составной код: через двоеточие, например `004.8:004.9`.

## 3. РЕФЕРАТ

### 3.1 Формульная строка (первая строка реферата)

```
Отчёт XX стр., X часть, XX рис., XX табл., XX источн., X прил. «НАИМЕНОВАНИЕ РАБОТЫ».
```

**Правила заполнения:**
- Все числа проставляются **после** полного завершения документа
- Наименование — дословно как на титульном листе
- Табл. и рис. — только из основной части (не из приложений, если не оговорено)

### 3.2 Ключевые слова

```
Ключевые слова: СЛОВО1, СЛОВО2, СЛОВО3, ...
```

- Только **ПРОПИСНЫМИ** буквами
- Через запятую
- 5–10 слов
- Отражают предметную область, применённые технологии и метод

### 3.3 Основные строки реферата (шаблон)

```
Объектом исследования являются [процессы / методы / механизмы ...].

Предметом исследования являются [модели / алгоритмы / программные средства ...].

Цель работы — [инфинитив: разработка и исследование / создание / проектирование ...].

В процессе работы проводились: [глагол+существительное через точку с запятой];
[глагол+существительное]; [глагол+существительное].

В результате исследования разработана/создана/исследована [что именно].

Степень внедрения — [опытная эксплуатация / промышленная эксплуатация / не внедрено].

Область применения — [конкретные пользователи и задачи применения].
```

## 4. СОДЕРЖАНИЕ

- Заголовок `СОДЕРЖАНИЕ` → стиль Word **Heading 1**
- Тело — поле автоматического оглавления Word:
  - Инструкция поля: `TOC \o "1-2" \h \z \u`
  - Охватывает уровни Heading 1 и Heading 2
- В содержании должны быть упомянуты **все** структурные элементы включая приложения
- После правок документа: обновить поле целиком (`F9` или ПКМ → «Обновить поле» → «Обновить целиком»)

## 5. ПЕРЕЧЕНЬ ОБОЗНАЧЕНИЙ И СОКРАЩЕНИЙ

- Заголовок `ОБОЗНАЧЕНИЯ И СОКРАЩЕНИЯ` → стиль **Heading 1**
- Формат каждой строки: `АББРЕВИАТУРА – расшифровка строчными буквами.`
- Разделитель — тире (`–`), **не дефис**
- Порядок: сначала латиница (алфавитно), затем кириллица (алфавитно)

**Пример:**
```
AI – искусственный интеллект.
API – интерфейс прикладного программирования.
REST – архитектурный стиль взаимодействия веб-сервисов.
БД – база данных.
```

## 6. ВВЕДЕНИЕ

Заголовок `ВВЕДЕНИЕ` → стиль **Heading 1**.

Строгая последовательность абзацев:

### 6.1 Актуальность (1–2 абзаца)

Объяснить **почему задача важна** и **какая проблема решается**. Шаблон:
```
Актуальность работы обусловлена [причина 1] и [причина 2]. Традиционные [аналоги]
требуют [недостатки]. В связи с этим востребованы [что именно нужно].
```

### 6.2 Цель НИР (1 предложение)

```
Цель НИР — [инфинитив], обеспечивающей/позволяющей [что именно].
```

### 6.3 Задачи (маркированный список)

```
Для достижения цели поставлены следующие задачи:
– [инфинитив] [что именно];
– [инфинитив] [что именно];
– [инфинитив] [что именно];
– [инфинитив] [что именно];
– [инфинитив] [что именно].
```

- Маркер — тире `–` (не `-`)
- Стиль абзацев — **List Paragraph**
- 5–7 задач, каждая начинается с глагола-инфинитива
- Точка с запятой в конце каждой, точка у последней

### 6.4 Методы исследования (1 предложение + ссылки)

```
Методы исследования: системный анализ, [метод 2], [метод 3], [метод 4] [n, m].
```

Ссылки на литературу **обязательны** уже во Введении.

## 7. ОБЩИЕ ПРАВИЛА ОФОРМЛЕНИЯ ОСНОВНОЙ ЧАСТИ

### 7.1 Нумерация заголовков

| Уровень | Пример | Стиль Word | Регистр |
|---|---|---|---|
| Раздел | `1 НАЗВАНИЕ РАЗДЕЛА` | Heading 1 | ПРОПИСНЫЕ |
| Подраздел | `1.1 Название подраздела` | Heading 2 | Первая прописная |
| Пункт | `1.1.1 Название пункта` | Heading 3 | Первая прописная |

- Заголовки — **без точки** в конце
- Между номером и названием — **пробел**, не точка
- Заголовки не переносятся; перед заголовком — разрыв страницы (для Heading 1)

### 7.2 Маркированные списки

```
Текст вводного предложения:
– элемент списка первый;
– элемент списка второй;
– элемент списка третий.
```

- Маркер `–` (тире), стиль абзаца **List Paragraph**
- Строчные буквы в начале каждого элемента
- Точка с запятой в конце каждого, точка в конце последнего
- Перед списком — двоеточие в конце вводного предложения

### 7.3 Нумерованные списки

```
Система обеспечивает:
1) шаг первый;
2) шаг второй;
3) шаг третий.
```

- Формат `N)`, **не** `N.`
- Те же правила знаков препинания, что у маркированных

### 7.4 Ссылки на литературу в тексте

| Тип | Пример |
|---|---|
| Одиночная | `[7]` |
| Множественная | `[7, 16]` |
| Диапазон | `[4–8]` |
| Смешанная | `[1–3, 11–15]` |

**Правила:**
- Квадратные скобки непосредственно перед точкой: `...текст [n].`
- Тире в диапазоне — `–` (не дефис, не `--`)
- Ссылка при **первом упоминании** каждой технологии/метода/инструмента
- Ссылки обязательны в **каждом разделе**

### 7.5 Ссылки на рисунки и таблицы

```
Компонентная диаграмма системы представлена на рисунке 2.3.1.
Весовые коэффициенты категорий приведены в таблице 2.1.
```

- До рисунка/таблицы — обязательная ссылка в тексте
- Нумерация — в пределах раздела: `Рисунок N.M.K`

**Подпись рисунка** — по центру **под** рисунком:
```
Рисунок 2.3.1 — Название рисунка
```

**Заголовок таблицы** — **над** таблицей:
```
Таблица 2.1 — Название таблицы
```

## 8. РАЗДЕЛ 1: АНАЛИЗ ПРЕДМЕТНОЙ ОБЛАСТИ И ПОСТАНОВКА ЗАДАЧИ

### 8.1 Подраздел 1.1 — Особенности задачи

Структура подраздела:
1. Описание специфики и сложности предметной области (2–3 абзаца)
2. Требования к разрабатываемой системе — маркированный список
3. Ссылки на литературу по предметной области `[n, m]`
4. Диаграммы (если есть): Use Case, доменная модель, BPMN-процесс

**Нумерация рисунков:** `Рисунок 1.1.1`, `Рисунок 1.1.2`, `Рисунок 1.1.3`

### 8.2 Подраздел 1.2 — Функциональная постановка задачи

Шаблон:
```
Система должна обеспечивать следующий функциональный цикл:
1) [действие];
2) [действие];
...
N) [действие].
```

## 12. ЗАКЛЮЧЕНИЕ

Заголовок `ЗАКЛЮЧЕНИЕ` → стиль **Heading 1**.
Ровно **3 абзаца**, строго в таком порядке:

```
Абзац 1 (итог работы):
В рамках НИР решена задача [что]. Реализованы [перечисление через запятую].

Абзац 2 (достижение цели + ссылки):
Поставленная цель достигнута: создано программное решение, позволяющее [что] [n–k].

Абзац 3 (практическая значимость):
Практическая значимость работы состоит в возможности использования системы
в задачах [конкретных задачах] [пользователями].
```

## 13. СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ

Заголовок `СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ` → стиль **Heading 1**.

### 13.1 Общие правила

- 20 источников (допустимо 15–25): **10 русских + 10 иностранных**
- Нумерация по **возрастанию**: `1.`, `2.`, ... `20.`
- Каждый источник — отдельным абзацем, стиль **Normal**

### 13.2 Форматы записей

**Нормативный документ (ГОСТ):**
```
N. ГОСТ X.XX-XXXX. Полное название стандарта.
```

**Книга (русскоязычная):**
```
N. Фамилия И.О. Название книги. Город: Издательство, Год. XXX с.
```

**Книга на иностранном языке:**
```
N. Surname I. Title of the Book. City: Publisher, Year. XXX p.
```

**Электронный ресурс:**
```
N. Название ресурса [Электронный ресурс]. URL: https://адрес/ (дата обращения: ДД.ММ.ГГГГ).
```

### 13.3 Рекомендуемый состав 20 источников

| Позиции | Тип | Примеры |
|---|---|---|
| 1–3 | ГОСТы | ГОСТ 7.32-2017; ГОСТ Р 7.0.100-2018; профильный ГОСТ |
| 4–10 | Русские книги по предметной области и технологиям | монографии, учебники |
| 11–15 | Документация применённых технологий | официальные сайты/docs |
| 16–20 | Иностранные книги (архитектура, алгоритмы, методы) | Fowler, Martin, Sommerville |

## 14. ПРИЛОЖЕНИЯ

### 14.1 Оформление заголовка каждого приложения

```
ПРИЛОЖЕНИЕ А          ← стиль Heading 1
(обязательное)        ← стиль Normal, по центру
Название приложения   ← стиль Normal, полужирный, по центру
```

Статус: `(обязательное)` или `(справочное)`.

### 14.2 Нумерация внутри приложений

| Тип | Формат |
|---|---|
| Рисунки | `Рисунок А.1 — Название` |
| Таблицы | `Таблица А.1 — Название` |
| Разделы | `А.1 Название` |

### 14.3 Приложение «Инструкция по эксплуатации» (Приложение А)

```
Вводный абзац: «В приложении приведена краткая инструкция по эксплуатации...»

Шаг 1. [Описание действия].
Шаг 2. [Описание действия].
...
Шаг N. [Описание действия].

Этап 1 иллюстрирован на рисунке А.1.
[место для скриншота]
Рисунок А.1 — Название экрана

Этап 2 иллюстрирован на рисунке А.2.
...
```

Рекомендуемые этапы для скриншотов:
1. Главная страница / вход
2. Создание основного объекта
3. Загрузка/импорт данных
4. Ключевая операция системы
5. Просмотр результатов
6. Экспорт / отчёт

### 14.4 Приложение «Программный код» (Приложение Б)

- Шрифт: **Courier New, 10 pt**
- Каждый файл предваряется строкой: `Файл: путь/к/файлу`
- Блок кода обрамляется тройными обратными кавычками (или без, если DOCX)
- Файлы располагаются в алфавитном порядке по пути
- Исключать: `node_modules`, `.venv`, `__pycache__`, `dist`, `build`, `.idea`

Включать:
- Серверная часть: `backend/app/` (модели, схемы, роутеры, сервисы, конфиги)
- Клиентская часть: `frontend/src/`
- Конфиги запуска: `docker-compose.yml`, `Dockerfile`, `requirements.txt`, `package.json`

## 15. СТИЛИ MICROSOFT WORD — СООТВЕТСТВИЕ ЭЛЕМЕНТОВ

| Элемент документа | Стиль Word |
|---|---|
| Структурные заголовки (ВВЕДЕНИЕ, ЗАКЛЮЧЕНИЕ и др.) | `Heading 1` |
| Нумерованные разделы (`1 НАЗВАНИЕ`) | `Heading 1` |
| Подразделы (`1.1 Название`) | `Heading 2` |
| Пункты (`1.1.1 Название`) | `Heading 3` |
